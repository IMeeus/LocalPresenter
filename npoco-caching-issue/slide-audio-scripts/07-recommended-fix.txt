The recommended fix is Option one: a non-generic static class called NPocoFactoryCompilationGuard, holding a SemaphoreSlim initialised to one and a ConcurrentDictionary that tracks which types have been compiled. Fulfill uses double-checked locking. The fast path is a cheap dictionary lookup — if the type is already compiled, we return immediately, no locking, no overhead. If it's not compiled yet, we acquire the semaphore, check again inside the lock to handle the case where another thread just compiled it, then run FulfillInternal, mark the type as compiled, and release the semaphore. After the first deploy, once every type has been compiled through this path, the semaphore is never touched again. One critical implementation detail: this guard class must be non-generic. If you put the statics on the generic BaseRequestStateManager, each closed generic instantiation gets its own separate lock instance, which means compilations of different types are no longer serialised at all.
