Here's the race. Thread A is compiling the factory for AbsenceRequestV3. Thread B is simultaneously compiling the factory for ClockingRemovalRequestV3. Both threads read the current list length - let's say it's five - before either one has called Add. Thread A bakes index five into its IL for the user reference converter. Thread B also bakes index five into its IL for the workflow reference converter. Then both threads call Add. Whatever lands at index five depends on scheduling. Thread B's factory now permanently applies the wrong converter: it reads an integer from the database, runs it through the user converter instead of the workflow converter, and when the generated IL tries to cast the result to a workflow reference, it throws an InvalidCastException. That broken factory is then cached.
