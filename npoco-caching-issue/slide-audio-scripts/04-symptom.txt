The first sign something is wrong is an InvalidCastException buried inside NPoco's generated factory code — a method called poco_factory_0. The message reads something like "unable to cast AggregateReference of IWorkflow and int to IAggregateReference of IUser and int". Sometimes it's a NullReferenceException from the same location. What makes this especially frustrating is that it only fires during rolling deploys — specifically when a fresh process receives its first wave of traffic. And once it fires, it's sticky: every subsequent request for that type throws the same exception. A process restart clears it, which can make it look like a flaky transient error when it's actually a deterministic corruption.
