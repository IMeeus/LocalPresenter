The investigation path is part of what makes this bug interesting. The alert shows up in Grafana pointing to a method called poco_factory_0. The problem is that poco_factory_0 is generated code — it lives inside the NPoco package, not in the application's own source. For years, the response was simply to redeploy: the alert fired, someone restarted the service, and the error went away. Nobody knew why. The breakthrough came from analysing the one place where all request types converge — BaseRequestStateManager.Fulfill. By tracing from that call site into NPoco's internals, the root cause was finally identified.
