Before diving into the technical details, it's worth understanding the cost. This bug has lived undetected in the codebase for years. Each time it fires, it causes widespread exceptions for all users on the affected web server. Because the system is event-sourced and these flows are not fully transactional, an exception mid-event leaves a request in a broken state: the event is persisted, but the state changes and side effects for that event are rolled back. The request finds itself in an inconsistent, half-applied state that can no longer receive any new actions — and it stays that way even after the bug is resolved. Team Bravas has been responsible for fixing these stuck requests by replaying the failed events, often in bulk across hundreds or thousands of affected requests per incident. It costs them significant time that could otherwise be spent on features. In the past month alone, this happened five times — and the frequency is increasing.
